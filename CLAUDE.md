# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## Commands

Always use `bunx` instead of `npx` — this is a Bun-only project.

### Development

```bash
# Type-check frontend + start Vite dev server (port 1420)
bun run dev

# Type-check frontend only
bun run typecheck

# Type-check sidecar server only
bun run --cwd server typecheck

# Run full Tauri app (also builds sidecar binary first)
bun run tauri dev

# Start sidecar server in watch mode (standalone)
bun run server:dev
```

### Building

```bash
# Build frontend to dist/
bun run build

# Build sidecar binary (Linux x64)
bun run server:build

# Build sidecar binary (Windows)
bun run server:build:windows

# Full Tauri build (Windows, includes sidecar + frontend + installer)
bun run build:windows

# Rebuild overlay assets (required when overlay HTML/CSS/JS changes)
bun run --cwd server bundle-assets
```

## Architecture

Three-layer stack: **Rust (Tauri)** ↔ **Bun/Elysia sidecar** ↔ **React frontend + browser overlay**

```
Twitch (EventSub WebSocket + Helix API + OAuth)
        │ stdout (bid JSON)
        ▼
Bun/Elysia Sidecar (port 3000)
  ├── EventSub Gateway (channel points + chat commands)
  ├── OAuth handler (/auth/*)
  ├── SSE broadcaster → Browser Overlay
  ├── /api/rewards, /health
  └── /overlay/* (7 widget types)
        │ stdin (overlay state, events)
        ▼
Tauri App (Rust)
  ├── Commands: start/pause/finish auction, approve/reject bids
  ├── State management + persistence
  └── Sidecar spawning/lifecycle
        │ Tauri events
        ▼
React Frontend (port 1420)
  ├── Pages: Setup, Live, Results, Overlays, Debug
  └── Zustand store
```

### Layer 1: Tauri Desktop App (`src-tauri/src/`)

- `lib.rs` — app entry; registers all Tauri commands; spawns sidecar on setup, kills it on exit
- `commands.rs` — all `#[tauri::command]` handlers:
  - `pick_image` — native file dialog → base64 data: URL
  - `open_url` — xdg-open (Linux) / powershell (WSL fallback)
  - `get_auction_state` — returns full AuctionData from mutex
  - `start_auction(config)` — sets Running, starts async timer loop, emits events
  - `pause_auction` — toggles Running ↔ Paused
  - `finish_auction` — sets Finished, returns winner bid
  - `approve_bid(id)`, `reject_bid(id)` — update bid status, push overlay state
  - `add_manual_bid(username, amount)` — creates Bid with source=Manual, auto-approves if not flagged
  - `set_widgets_show_after_finished(ids)` — updates which overlay widgets appear after auction ends
- `state.rs` — data models: `AuctionData`, `Bid`, `AuctionConfig`, `AuctionStatus`, `BidStatus`, `BidSource`
- `sidecar.rs` — spawns Bun binary as child process; reads stdout for incoming bids; writes JSON to stdin

Auction state is persisted to `~/.config/twitch-auction/auction-state.json`.

#### Bid Flagging Logic (in `commands.rs`)
A bid is flagged (set to Pending, requiring manual approval) if:
- `amount > config.max_bid_threshold` (when set), OR
- `amount > current_max_approved_bid × config.unrealistic_multiplier`

Non-flagged bids are auto-approved immediately.

### Layer 2: Bun/Elysia Sidecar (`server/src/`)

- `index.ts` — Elysia app (port 3000); reads JSON from Rust stdin via line-buffered reader; routes HTTP and SSE
- `twitch/gateway.ts` — Twitch EventSub WebSocket:
  - Deletes stale subscriptions on startup (polls until cleared, max 15s)
  - `onChannelRedemptionAdd` → bid with source `channel_points`; fetches + caches user avatar
  - `onChannelChatMessage` → watches for `!bid <amount>` command; bid with source `chat_command`
  - Avatar URLs cached in-memory (`Map<userId, url>`) to avoid repeated API calls
  - Filters: if `config.reward_id` set, only accept redemptions matching that reward ID
  - If `config.disable_chat_when_reward=true` and a reward is configured, blocks chat `!bid`
- `twitch/auth.ts` — OAuth 2.0 token exchange; fetches user info; saves to `state.json`
- `twitch/refund.ts` — channel points refund logic for rejected bids
- `overlay/sse.ts` — SSEBroadcaster: manages connected clients; caches last `auction_state` and sends it immediately to new connections
- `overlay/assets.ts` — serves overlay files via Bun file API
- `overlay/assets-inline.ts` — auto-generated by `bundle-assets`; all HTML/CSS/JS inlined as base64
- `routes/overlay.ts` — wildcard routes for `/overlay/*` + SSE at `/overlay/events`
- `routes/rewards.ts` — `GET /api/rewards` → Twitch Helix custom rewards
- `routes/auth-callback.ts` — `GET /auth/status`, `GET /auth/callback?code=...`
- `routes/health.ts` — health check + uptime
- `settings.ts` — reads/writes `~/.config/twitch-auction/state.json` (tokens, channel info) and `server/config.json` (Twitch credentials, overlay defaults)
- `logger.ts` — in-memory log buffer (max 300 entries); exposed at `GET /logs` for Debug page

Bundled overlay assets live in `server/overlay/assets-inline.ts` (base64-encoded CSS/JS).

### Layer 3: React Frontend (`src/`) + Browser Overlay (`server/overlay/`)

- `src/App.tsx` — **registers all Tauri event listeners here, once, in a single `useEffect`** (never in hooks or child components — causes duplicate events); handles routing between pages based on `AuctionStatus`
- `src/store/auctionStore.ts` — Zustand store; single source of truth for UI state
- `src/pages/Setup.tsx` — idle state: Twitch auth button, `<AuctionSetup>` config form, Widgets button
- `src/pages/Live.tsx` — running/paused: `<LotPreview>`, Timer, `<BidList>` with filter tabs, `<ManualBidForm>`, Finish button
- `src/pages/Results.tsx` — finished: winner card, overlay widget URL, approved bids list, New Auction button
- `src/pages/Overlays.tsx` — preview all 7 overlay widgets; toggle "show after finished" per widget; open/copy URLs
- `src/pages/Debug.tsx` — server status, uptime, real-time log viewer (auto-scroll, color-coded levels)
- `src/components/AuctionSetup/` — large config form; persists to localStorage key `auction-setup-config`
- `src/components/BidList/` — sortable/filterable bid list with approve/reject buttons
- `src/components/Timer/` — MM:SS display with play/pause; local decrement via `useTimer`
- `src/components/LotPreview/` — lot image (`data:` URL or Tauri `asset:` scheme) + title + price
- `src/components/ManualBidForm/` — username + amount form → `add_manual_bid` Tauri command
- `src/components/FadeImage.tsx` — `<img>` wrapper with opacity fade-in on load
- `src/hooks/useAuction.ts` — wraps all Tauri command invocations
- `src/hooks/useTwitchAuth.ts` — OAuth flow; polls `/auth/status` every 2s (max 60 attempts)
- `src/hooks/useRewards.ts` — fetches `/api/rewards`; handles `channel_points_not_available` error
- `src/hooks/useTimer.ts` — decrements `timer_left_seconds` locally every 1s while Running
- `src/hooks/useServerStatus.ts` — polls `/health`; returns `connected|disconnected|checking`
- `src/hooks/useServerLogs.ts` — fetches `/logs` with optional auto-refresh

**Overlay widgets** (`server/overlay/`): 7 standalone HTML+JS+CSS scenes for OBS/Streamlabs:

| Widget ID | Dimensions | Description |
|-----------|-----------|-------------|
| `overlay` | 1920×500 | Full scene (lot + price + timer + bids) |
| `lot` | 420×420 | Lot image + title |
| `price` | 460×180 | Starting price |
| `timer` | 380×110 | Countdown |
| `bids` | 560×600 | Top bids list |
| `top-bid` | — | Current leader |
| `winner` | — | Final winner |

`server/overlay/widgets.tsx` — React component library shared between overlay demos and the Overlays page preview; exports `WIDGET_DIMENSIONS` record used for scaling previews in the UI.

Browser overlay served at `http://127.0.0.1:3000/overlay`, uses SSE for real-time updates.

### Communication Channels

**Tauri events (Rust → React):** `bid:new`, `bid:updated`, `timer:tick`, `auction:started`, `auction:status`, `auction:finished`

**Sidecar stdin/stdout (Rust ↔ Bun):**
- Rust → Bun: `{ type: "overlay:state" | "bid:approved" | "bid:rejected" | "timer:tick" | "auction:finished", data: {...} }`
- Bun → Rust: `{ type: "bid", username, user_id, amount, source, redemption_id?, reward_id?, avatar_url? }`

**SSE (Bun → browser):** `auction_state`, `bid_approved`, `bid_rejected`, `timer_tick`, `auction_finished`

**HTTP (React → Bun):** `GET /health`, `/auth/status`, `/auth/callback`, `/api/rewards`, `/overlay/events`, `/overlay/*`, `/logs`

### Persistence

| File | Contents |
|------|----------|
| `~/.config/twitch-auction/auction-state.json` | Full AuctionData (written after every state change) |
| `~/.config/twitch-auction/state.json` | OAuth tokens + channel login/ID |
| `server/config.json` | Twitch client_id/secret, overlay defaults, auction defaults |

### Key Data Models (`src-tauri/src/state.rs` + `src/store/auctionStore.ts`)

**AuctionConfig** notable fields:
- `currency`: `"channel_points"` | `"custom"` + `currency_label`
- `reward_id?`: if set, only accept channel point redemptions for this reward
- `disable_chat_when_reward`: block `!bid` chat command when a reward is configured
- `min_bid_step` / `min_bid_step_percent?`: absolute or percentage-based minimum raise
- `unrealistic_multiplier` + `max_bid_threshold?`: bid flagging thresholds
- `snipe_protection_seconds`: auto-extend timer when bid arrives near end
- `top_bids_in_overlay`: number of bids shown in overlay bids widget
- `widgets_show_after_finished: string[]`: widget IDs to display when auction ends

**BidSource:** `channel_points` | `chat_command` | `manual`
**BidStatus:** `pending` | `approved` | `rejected` | `winner`

All UI text is in **Russian**. Number formatting uses `toLocaleString('ru-RU')`.

### Known Gotchas

- **Stdin buffering:** Bun stdin delivers data in chunks. The sidecar accumulates lines in a buffer before calling `JSON.parse` — never split and parse individual chunks, or large payloads (e.g. base64 images in `overlay:state`) will fail silently.
- **Sidecar binary path:** The compiled binary must be at `src-tauri/binaries/auction-server-<target>` for Tauri to bundle it. The `server:build` scripts handle this.
- **Overlay assets:** Run `bun run --cwd server bundle-assets` before building if overlay HTML/CSS/JS changes.
- **Event listeners:** Register Tauri `listen()` calls only in `App.tsx` useEffect. The hook `useAuction` is consumed by multiple components simultaneously — registering listeners there causes duplicate bid events.
- **SSE caching:** `SSEBroadcaster` caches the last `auction_state` and sends it immediately to new connections. New overlay logic must update the cache on any state change.
- **Subscription cleanup:** On sidecar startup, all stale EventSub subscriptions are deleted (polls up to 15s). Avoid assuming subscriptions persist between runs.
- **WSL open_url:** `open_url` tries `xdg-open` first, falls back to `powershell.exe Start-Process` for WSL environments.
- **Tauri `asset:` scheme:** Lot images from disk are served via `convertFileSrc()` / the Tauri asset protocol. `data:` URLs (from the image picker) are used directly without conversion.
